# OAuth 2.0 Client Credentials Grant - Very Brief

## **Flow Steps:**

1. **Client** sends `client_id` and `client_secret` to Apigee
2. **Apigee** validates credentials and issues access token directly
3. **Client** uses access token to access APIs

## **Apigee Policy:**

```xml
<OAuthV2 name="GenerateAccessToken">
  <Operation>GenerateAccessToken</Operation>
  <GrantType>client_credentials</GrantType>
</OAuthV2>
```

## **Key Features:**
- **Simplest OAuth flow** - only 2 steps
- **No user involvement** - machine-to-machine
- **No refresh token** - client re-authenticates when token expires
- **Uses client credentials** directly for authentication

## **Use Cases:**
- Server-to-server communication
- API microservices
- Background processes
- No user context needed
