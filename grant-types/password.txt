# OAuth 2.0 Password Grant (Resource Owner Credentials) - Very Brief

## **Flow Steps:**

1. **Client** sends user's `username` and `password` to Apigee
2. **Apigee** validates credentials and issues access token directly
3. **Client** uses access token to access APIs

## **Apigee Policy:**

```xml
<OAuthV2 name="GenerateAccessToken">
  <Operation>GenerateAccessToken</Operation>
  <GrantType>password</GrantType>
</OAuthV2>
```

## **Key Features:**
- **Direct credential exchange** - username/password for token
- **Single request** for token
- **Returns both access + refresh tokens**
- **Requires highly trusted clients**

## **Use Cases:**
- First-party trusted applications
- Mobile apps from same organization
- Legacy system migrations

## **Security Note:**
- **Least secure** OAuth flow
- Only use with **highly trusted clients**
- Avoid for third-party applications
